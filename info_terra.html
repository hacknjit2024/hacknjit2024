Project Overview

We have created a steampunk-themed library website hosted on AWS, with content served via CloudFront. This project involves:

  • An S3 bucket to store static assets, such as index.html and images.
  • A CloudFront distribution to serve the content with low latency and HTTPS.

Why Terraform?

Terraform is an Infrastructure as Code (IaC) tool that allows us to define and deploy cloud infrastructure in a repeatable and automated way. Here’s how it benefits our project:

  • Automation and Reproducibility: With Terraform, we can set up and configure AWS resources like S3 buckets, CloudFront distributions, and bucket policies with code. This makes it easy to redeploy or modify infrastructure, ideal for the fast pace of a hackathon.
  • Version Control and Collaboration: Since infrastructure is defined in code, we can track changes, collaborate, and roll back if needed.
  • Scalability: Using Terraform means we can easily expand this infrastructure, such as adding more S3 buckets or caching rules, as the project grows.

What We Used Terraform For in This Project

Here are the specific resources we provisioned with Terraform:

  • S3 Bucket: To store the static files (HTML, images) for the website, with public access set through a bucket policy.
  • CloudFront Distribution: To serve content globally, providing faster access with caching and HTTPS.
  • Bucket Policies and Permissions: Managed with Terraform to ensure that objects in the bucket are publicly accessible.
  • Infrastructure Management: By running simple Terraform commands, we can create, modify, or tear down the entire setup as needed.

How Terraform Improved Efficiency

  • Quick Setup: We could quickly create all required resources in one go without manually configuring each one.
  • Consistency: Terraform ensured consistent infrastructure setup every time we applied changes, avoiding manual errors.
  • Easy Changes: During the hackathon, if we needed to adjust a setting (like changing a bucket policy or cache behavior), we could update the Terraform configuration and reapply.

How It Works Together

This is how Terraform, S3, and CloudFront integrate in our project:

  • Terraform provisions and configures both S3 and CloudFront.
  • The S3 bucket hosts static files, while CloudFront caches and distributes them globally.
  • The combination allows us to deliver a website with low latency, security, and scalability.
